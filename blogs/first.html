    不得不知道的加密方式🔥

    Fan Yiming 2023/1/1 JavaScript手写

    ​ SM4加密算法是中国国家密码管理局于2012年发布的一种分组对称加密算法，也是中国政府推荐的商用密 码算法之一。以下是SM4有一下几个特点：

    • 安全性高：SM4算法采用128位密钥长度和128位的分组长度，通过多轮迭代和异或运算，增强了其抗密码分析的能力。算法经过了广泛的安全性评估和密码学社区的评审，被认为是安全可靠的加密算法。
    • 隐私保护：SM4算法可用于保护个人隐私和敏感数据的安全传输和存储。它可以在无线通信、电子支付、云计算等场景中使用，确保数据在传输和存储过程中不被未授权的人读取或篡改。
    • 国家安全：作为中国自主研发的密码算法，SM4加密算法对于国家信息安全和国家网络基础设施的保护具有重要意义。它提供了一种安全、高效的加密解决方案，可用于政府和军事部门的敏感信息保护。
    • 兼容性广泛：SM4加密算法在软硬件实现上相对简单，并且在国内外得到了广泛支持和应用。许多主流的加密软件和硬件设备已经支持SM4算法，使其更易于集成和应用于各种系统和平台。

    • 在接口请求过程中，使用解密后的SM4密钥 对入参进行加密，此过程在请求拦截器中进行

      function reqInterceptor(request) {
 	const { url } = requset   // 获取请求的接口
 	// 获取需要加密的url列表
 	const smList = Store.getters['account/smList']
 	if (smList.length > 0 && smList.includes(dealUrl(url))) {
 	
        // 获取前端私钥
        const privateKeyBefore = Store.getter['account/jsPrid']

        // 去除后端加解密前缀 '04' 再进行解密
        const decryptSm4 = Store.getters['account/sm4Pid'] ? Store.getters['account/sm4Pid'].slice(2,  Store.getters['account/sm4Pid'].length) : null

        // 获取传递标识的值
        const transferId = Store.getters['account/ transferId']

        // 获取加密用的SM4密钥的值
        const SM4KEY = transferId ? sm2.doDecrypt(decryptSm4, privateKeyBefore) : Store.getters['account/Sm4Pid']

        // 同时设置最终加密所使用的密钥 （解密后的密钥或者是无需解密的密钥）
        Store.commit('aaccount/SET_DECREPTSM4', SM4KEY)

        // 加密入参
        request.data = request.data ？ { data: sm4.encrypt(JSON.stringfy(request.data), SM4KEY)} : null

 	}
 	
}

      这里使用 dealUrl 方法又对接口进行了处理 以满足接口路径和后端返回的url列表格式吻合

function dealUrl () {
	const regex = /([^?+])/   // 首先利用正则将接口中携带的参数去除
	
	// 检查是否存在 '?' 存在意味存在参数
	if (api.includes('?')){
	  return api.match(regex)[1].replace(/^(?!\/)/, '/')
	} else {
	  return api.replace(/^(?!\/)/, '/')
	}
}
